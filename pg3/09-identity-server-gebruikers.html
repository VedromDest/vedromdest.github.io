User Accounts Activeren in Identity Server

Identity Server biedt een implementatie van OpenID Connect die we kunnen gebruiken om user auth te doen.

Om mensen door de login flow te loodsen hebben we schermen nodig.Je kan deze volledig zelf ontwikkelen of vertrekken van de Identity Server templates.

Gezien dit een backend cursus is, zullen we de ingebouwde ui-templates gebruiken.

src\ShipIt\ShipIt.IdentityServer
dotnet new isui

De nodige calls om de user interface te activeren staan al in de code, maar moeten uit commentaar gezet worden.

ShipIt.IdentityServer\HostingExtensions.cs
public static WebApplication ConfigureServices(this WebApplicationBuilder builder)
{
    // uncomment if you want to add a UI
    builder.Services.AddRazorPages();
    ...
}

public static WebApplication ConfigurePipeline(this WebApplication app)
{
    ...
	// uncomment if you want to add a UI
	app.UseStaticFiles();
	app.UseRouting();
	...
	// uncomment if you want to add a UI
	app.UseAuthorization();
	app.MapRazorPages().RequireAuthorization();
	...
}

1 Injectie van de test users.

Het kan interessant zijn de definities van Bob en Alice van naderbij te bekijken.

Start het IdentityServer project op en surf naar https://localhost:5001/Account/Login. Log in met een van de test users bob/bob of alice/alice.

Na het inloggen verschijnt bovenaan in de navigatiebalk een menu item dat dient als bevestiging dat je ingelogd bent. Je kan ook uitloggen.

De user interface van Identity Server is nu operationeel. We kunnen deze users gebruiken in een eigen front-end applicatie.

Voor de scope van dit vak zijn de test users voldoende. Indien je interesse hebt om een realistisch front-end project te bouwen met een user subscription flow en bijhorende database, kan je je verdiepen in ASP.NET Core Identity en de samenwerking met IdentityServer, alsook claims based authorization.
