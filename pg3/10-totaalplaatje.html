Rudimentaire ShipIt Web App

De UI van Identity Server is bedoeld om de login flow van een user te faciliteren. Het is niet de bedoeling in de IS web app je business logica en web pagina’s toe te voegen.

We maken een nieuwe web app voor de business flows. Deze web app zal met Identity Server spreken en dmv redirect de user naar de login pagina brengen. Na succesvolle login zal deze terug naar de business app geleid zal worden.

De business app zal in de achtergrond tegen de applicatie-api’s spreken.

.Net Project

Voeg een Web App met Razor Pages toe aan de Solution.

Start de nieuwe web app eens op om te valideren dat alles correct aangemaakt werd.

App Client

De nieuwe web app heeft zelf een client nodig om met IdentityServer te spreken. Voeg in IdentityServer een client toe voor deze web app. Let er op dat je het juiste poortnummer gebruikt van jouw web app. Dit kan verschillen van het voorbeeld en moet verschillen van de poort waarop IS zelf draait.

Config.cs
new Client {
    ClientId = "webapp-client",
    ClientSecrets = {new Secret("eenNogGroterGeheim".Sha256())},
    AllowedGrantTypes = GrantTypes.Code,
    AllowedScopes = {
        IdentityServerConstants.StandardScopes.OpenId,
        IdentityServerConstants.StandardScopes.Profile,
        "shipit.pricequotes.api.read" (1)
    },
    RedirectUris = { "https://localhost:7164/signin-oidc" },
    PostLogoutRedirectUris = { "https://localhost:7164/signout-callback-oidc" },
}

Deze Client krijgt toestemming om user profile claims op te vragen. We gaan de passende scope moeten activeren.

public static IEnumerable<IdentityResource> IdentityResources =>

Gezien Identity Server de Clients uit een database leest, en niet uit de in-memory config, zouden we de configuratie in de databank moeten toevoegen. Indien dit frequent nodig is in een bedrijf, zal men een UI voorzien.

Hier kiezen we ervoor de databaseinitialisatielogica voor Clients een beetje bij te sturen. Wanneer Clients enkel bestaan in Config, voegen we ze toe aan de database.

Verbinden Met Identity Server

Voeg nuget package Microsoft.AspNetCore.Authentication.OpenIdConnect 8.0.11 toe aan de ShipIt WebApp. Deze package, in combinatie met de nodige configuratie en net aangemakte Client, zal de web app laten inhaken op Identity Server.

ShipIt.WebApp/Program.cs
//...
builder.Services.AddAuthentication(options =>
{
	options.DefaultScheme = "Cookies";
	options.DefaultChallengeScheme = "oidc"; (1)
})
.AddCookie("Cookies")
.AddOpenIdConnect("oidc", options =>
{
	options.Authority = "https://localhost:5001";

	options.ClientId = "webapp-client";
	options.ClientSecret = "eenNogGroterGeheim";
	options.ResponseType = "code";

	options.Scope.Clear();
	options.Scope.Add("openid");
	options.Scope.Add("profile");

	options.GetClaimsFromUserInfoEndpoint = true;
	options.MapInboundClaims = false;

    // access_token en id_token bijhouden
	options.SaveTokens = true;
});
//...

ShipIt.WebApp\Pages\GetQuote.cshtml.cs
public class GetQuoteModel : PageModel
{
    private readonly IConfiguration _configuration;
    public string Json { get; set; } = string.Empty;
    public QuoteResponseDTO QuoteResponse { get; set; } = new QuoteResponseDTO();

    [BindProperty]
    public string QuoteId { get; set; }

    public GetQuoteModel(IConfiguration configuration)
    {
        _configuration = configuration;
    }

    public async Task OnPost()
    {
        var accessToken = await HttpContext.GetTokenAsync("access_token");

        var client = new HttpClient();
        client.SetBearerToken(accessToken);

        var response = await client
            .GetAsync($"{_configuration["QuoteApiBaseUrl"]}{QuoteId}");

        var stringContent = await response.Content.ReadAsStringAsync();
        Json = stringContent;

        var options = new JsonSerializerOptions();
        options.PropertyNameCaseInsensitive = true;
        options.Converters.Add(new JsonStringEnumConverter());
        QuoteResponse = await response.Content
            .ReadFromJsonAsync<PriceQuoteResponseContract>(options);
    }
}

We gebruiken een PriceQuoteResponseContract dus we maken een Reference naar het Contracts project.

Ook de IdentityModel nuget package wordt gebruikt.

We voegen de base url van de Quote Api toe in de appsettings.json.

Het access_token bevat de credentials om een backend api te roepen. Dit token bevat geen gedetailleerde info over de user. Je kan wel het id van de user vinden in de sub claim.

Ditzelfde token bevat ook de nodige rechten om bij het userprofile endpoint (https://host:port/connect/userinfo) van Identity Server meer details over de user op te vragen. Indien nodig kan de backend api daar gebruik van maken.

User Gebruiken In Backend

Laat ons de user uitlezen in de Quote api en Bob (id 2) rejecten.

public async Task<ActionResult<PriceQuoteResponseContract>> GetQuoteAsync([FromRoute]string quoteId)
{
    if(await GetUserIdIfAnyAsync() == "2")
        return Forbid();
    var quote = await _priceQuoteService.GetQuoteAsync(quoteId);
    return Ok(quote);
}

private async Task<string?> GetUserIdIfAnyAsync()
{
    var accessToken = await HttpContext.GetTokenAsync("access_token");
    var parsedToken = new JwtSecurityTokenHandler().ReadJwtToken(accessToken);

    var userId = parsedToken
        .Claims.FirstOrDefault(_ => _.Type == "sub")?.Value;

    if (string.IsNullOrWhiteSpace(userId))
        return null;

    return userId;
}
Je kan de userId opslaan bij data in het systeem en vergelijken bij het uitlezen om een mate van security te implementeren.
