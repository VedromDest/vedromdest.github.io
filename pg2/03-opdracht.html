Tussentijdse Evaluatie

Doel

Ontwerp en implementeer een beknopt systeem voor het beheer van voetbalploegen.

België - Japan op 2 juli 2018

Deliverables

  • MySQL DDL/DML script

  • Werkende dotnet solution

  • Postman collection die je gebruikt hebt om te testen

Requirements

Business

  • Een speler

    • heeft altijd een naam, voornaam, geboortedatum en uniek e-mail adres

    • kan lid zijn van één of meer ploegen

  • Een lidmaatschap is geldig

    • voor een bepaalde periode (datum van - datum tot)

      • datum van is verplicht

      • datum tot is niet altijd gekend

  • Een ploeg heeft

    • unieke naam

    • naam van de trainer

    • niveau: Regionaal, Nationaal, Internationaal

Een speler kan bij een specifieke ploeg maximaal 1 lidmaatschap hebben met onbekende einddatum.

Domein Model

Technisch

  • Gebruik MySQL

  • Gebruik Entity Framework Core scaffolding

  • Gebruik een drielagen backend architectuur

    • Api

      • gebruik aparte request en response contracten ~ DTO’s

    • Service ~ BL

    • Storage ~ DAL

Api

Spelers
POST /players      -> 1 maken
GET  /players      -> lijst van alle (1)
GET  /players/{id} -> 1 specifieke (1)
1 Het response contract bevat alle velden van een speler, inclusief het Id.
De lidmaatschappen worden niet bijgevoegd.
Lidmaatschappen
POST /memberships      -> 1 maken
GET  /memberships      -> lijst van alle (1)
GET  /memberships/{id} -> 1 specifieke (1)
PUT  /memberships/{id} -> update 1 specifieke (2)
1 Het response contract bevat ook naam en leeftijd van de speler én naam van de ploeg.
2 Enkel de einddatum mag aangepast worden, als deze nog niet gekend was.
Ploegen
POST /teams      -> 1 maken
GET  /teams/{id} -> 1 specifieke (1)
1 Bevat ook een lijst met de namen van de spelers.
